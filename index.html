string, math, array

Posted on 2021-04-18 In basic

总结string、math、array的常用方法

String

charAt()

  • 返回指定索引处的字符 stringObject.charAt(index)
  • index 必需，如果index 不在 0 与 string.length-1 之间，将返回一个空字符串
    var str="fighting 2018!";
str.charAt(3);//h

    indexOf()

  • 返回指定字符串在字符串中首次出现的位置 stringObject.indexOf(str,startpos)
  • str 必需，需要检索的字符串
  • startpos 可选的整数参数，规定开始检索的位置, 0 到length - 1
    var str="fighting 2018!";
str.indexOf("t");//4

split()

将字符串分割成字符串数组，并返回此数组 stringObject.split(separator,limit)
separator 必需，从该参数指定的地方分割。 limit 可选，分割的次数

"2:3:4:5".split(":")    //将返回["2", "3", "4", "5"]
"|a|b|c".split("|")    //将返回["", "a", "b", "c"]

substring()

提取两个指定下标之间的字符，从 start 处到 stop-1
stringObject.substring(start,stop)
start 必需
stop 可选

var str="Hello world!";
document.write(str.substring(3));//lo world!
document.write(str.substring(3,8));//lo wo

event loop

Posted on 2021-04-17 In basic

  1. define
    browser event loop: bridge between js and browser API through callback func
    node event loop: bridge between js and system through callback func

  2. process & thread
    process: programm
    thread: smallest unit of system scheduling
    open a programm means create a new process, including several threads
    the running one called main thread

browser event loop

async func will put into “event queue” through web api
event queue will put the async func into “call stack” and the async func will be envoked

2 types of event queue

macrotask queue:
ajax, setTimeout, setInterval, DOM listener, UI render
microtask queue:
promise.then(), Mutation Observer API, queueMicrotask

priority between event queue

  1. main script code will be executed at first
  2. before execute any marcotask, event loop will executes all of task in microtask queue

keypoint：

remeber all of the task
write down ma_queue, mi_queue, output

webpack-interview

Posted on 2021-04-13 In webpack

前端开发的复杂度和规模庞大，需要工程化来独立开发
如react的jsx代码必须编译后才能在浏览器中使用
如sass和less的代码浏览器也是不支持
如果摒弃了这些开发框架，那么开发的效率将大幅下降。在众多前端工程化工具中，webpack脱颖而出成为了当今最流行的前端构建工具。 然而大多数的使用者都只是单纯的会使用，而并不知道其深层的原理。

问题一览

webpack与grunt、gulp的不同？
与webpack类似的工具还有哪些？谈谈你为什么最终选择（或放弃）使用webpack？
有哪些常见的Loader？他们是解决什么问题的？
有哪些常见的Plugin？他们是解决什么问题的？
Loader和Plugin的不同？
webpack的构建流程是什么?从读取配置到输出文件这个过程尽量说全
是否写过Loader和Plugin？描述一下编写loader或plugin的思路？
webpack的热更新是如何做到的？说明其原理？
如何利用webpack来优化前端性能？（提高性能和体验）
如何提高webpack的构建速度？
怎么配置单页应用？怎么配置多页应用？
npm打包时需要注意哪些？如何利用webpack来更好的构建？
如何在vue项目中实现按需加载？

问题解答

1. webpack与grunt、gulp的不同？

轻量化的任务会用gulp来处理，比如单独打包CSS文件

grunt和gulp是基于任务和流（Task、Stream）的
类似jQuery，找到一个（或一类）文件，对其做一系列链式操作，更新流上的数据， 整条链式操作构成了一个任务，多个任务就构成了整个web的构建流程。

webpack是基于入口
webpack会解析入口所需要加载的所有资源文件，用不同的Loader处理文件，用Plugin来扩展功能

总结：
从构建思路来说
gulp和grunt将构建过程拆分成多个Task，并合理控制所有Task的调用关系
webpack需要入口，需要清楚对于不同的资源应该使用什么Loader做解析

gulp更像后端开发者的思路，需要对于整个流程了如指掌
webpack更倾向于前端开发者的思路

basic

Posted on 2021-04-13 In webpack

概念

  • webpack是js的静态模块打包工具
  • 处理js程序时会构建dependency graph，对应映射到项目的每个模块，生成多个bundle
  • webpack4不再引入配置文件来打包项目

入口entry

  • entry指示构建denpendency graph的开始
  • 进入entry后，webpack会找出与entry依赖的模块和库
  • 默认值是 ./src/index.js
  • 在webpack.config.js自定义entry
    module.exports = {
entry: 'xxx'
};

输出output

  • 定义输出bundle的位置和名称
  • 默认值是 ./dist/main.js，其他文件在./dist
  • 在webpack.config.js自定义output
    const path = require('path');
module.exports = {
entry: 'xxx',
output: {
  path: path.resolve(__dirname, 'dist'),
  filename: 'xxx'
}
};

loader

webpack只能理解js和json，loader让webpack处理其他类型文件并转为模块，添加到dependency graph

load的两个属性

  • test：识别文件
  • use：定义转换所用的loader
