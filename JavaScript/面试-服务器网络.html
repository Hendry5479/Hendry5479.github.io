              1、http、https

              http：超文本传输协议，80

              http：HTTP加上SSL层建立安全通道，443

              https更安全，需要ca证书费用高，相对低效

              为什么说https更安全？

              https的工作原理

              1、客户端要求建立ssl连接

              2、服务器返回证书，包含公钥

              3、双方协商ssl安全等级，建立会话密钥

              4、通过证书公钥加密会话密钥，客户端传送给服务器

              5、服务器用私钥解开会话密钥

              非对称加密和对称加密结合的方式

              可能有中间人攻击，中间交换成自己的公钥，获取到双方的内容

              证书校验公钥是否被篡改，证明证书有效，数字签名，ca提供的一种hash算法，秘钥加密签名，公钥解密签名，证明真伪

              http版本

              1.0：短连接，断开重新握手；头部阻塞

              1.1：长连接

              2.0：多路复用，头部压缩，分帧传输，服务器主动推送

              TCP

              三次握手

              浏览器发送syn = 1, seq = x的数据包

              服务器返回ack = 1, ack number = x + 1, syn = 1, seq = y

              浏览器检查返回的seq是否正确，ack是否为1， 然后发送ack = 1, ack number = y + 1

              为什么要三次握手？

              如果只有两次握手，服务器无法确定浏览器的接收能力，服务端无法确定浏览器是否获取到自己正确的序列号，可靠性无法保证

              避免重复建立连接：

              浏览器可能发送了两个不同序号的SYN包，三次握手的话，浏览器需要比对ACK和自己发送的SYN，旧的话发送RST，等待最新的返回才建立连接

              第三次握手可以携带数据包吗？

              可以，因为客户端已经确定服务端的收发能力，可以携带数据包。

              四次挥手

              客户端发送FIN=x给服务端，进入FIN-WAIT状态，停止发送

              服务端返回ACK=x+1，进入CLOSE-WAIT状态，停止接收

              服务端发送FIN=y，进入LAST-ACK

              客户端发送ACK=y+1，进入TIME-WAIT，过一段时间后再进入CLOSED；服务端收到后进入CLOSED

              为什么握手是三次，挥手是四次？

              因为握手时，服务端可以将ACK和SYN合并为一次发送。

              而挥手时，客户端关闭发送后，服务端可能还要发送数据，所以不能将FIN包和ACK包一起发，所以分为四次

              为什么TIME_WAIT 状态需要经过 2MSL 才能返回到 CLOSE 状态？

              MSL指的是报文在网络中最大生存时间

              客户端发送的ACK包可能收不到，需要等一个来回确认。
