              3、事件循环

              过程

              每次循环叫做一个tick

              执行栈选择宏任务script进入任务队列，执行所有同步代码

              然后检查是否有微任务，并清空微任务队列

              浏览器可能会渲染页面

              开始下一轮tick，执行宏任务中的异步代码，例如回调或者定时器

              vue nextTick就是在下一轮tick中微任务中执行传入的回调函数，基于事件循环实现的

              宏任务与微任务的区别

              es6中，宏任务叫task，微任务叫job

              宏任务由主机生成，微任务由js引擎生成；宏任务可以触发新的一轮tick

              微任务的意义

              在es3，js引擎本身无法生成异步请求，所以没有微任务

              在es5，promise被引入js，js引擎本身可以生成异步任务而不需要浏览器

              宏任务和微任务都可以异步操作，微任务解决了什么问题？

              主线程有可能执行大量任务，异步回调有可能等待很长时间，微任务的诞生是一种实现实时性和高效之间的平衡。

              8、v8垃圾回收

              两种GC方法（特点，缺点）

              V8GC（优化点）

              js引用类型保存在堆内存中，若没有了引用关系，则是不可达对象，需要被清理，否则会内存不足

              标记清除：活动对象打标记，清除阶段时销毁无标记对象

              ​ 引擎GC时从根节点（有多个）遍历内存中的对象打标记

              ​ 分配内存的三种策略：First-fit、Best-fit、Worst-fit

              ​ **特点：**简单，但是有内存碎片、慢

              ​ 标记整理法：标记结束后将活动对象移动到一端，然后清理

              引用计数法：被引用、赋给另一个变量时+1，被覆盖则-1；0则销毁

              ​ 缺点：1、循环引用无法解决；2、计数器占用内存大

              ​ A和B相互引用，则引用数量都是2，不被清理

              ​ 标记清除策略的话，在函数结束后，两个对象都不在作用域中，都被清除掉。

              V8的GC优化：基于标记清除的分代式GC

              新生代的特点是存活时间短、比较小；因此将内存区域区分为新老生代，使用不同频率的GC，优化性能

              新生代的GC

              ​ 将新生代的堆内存分为使用区和空闲区，标记一次后，将活动对象转去空闲区，然后清空使用区，反转

              ​ 如果多次GC仍然存活，或者对象比较大，则转去老生代

              老生代GC

              ​ 不用复制法，直接使用标记清除整理法

              并行回收

              Stop-The-World：全停顿，JS脚本等待GC后恢复运行的行为

              GC会造成stw，而并行回收则是在GC时引入辅助线程提高效率

              并行回收依然会停顿，但是会减少耗时，而且不用考虑引用关系的改变

              新生代GC就采用并行回收

              增量标记和懒清理

              并行回收依然全停顿，对于老生代而言耗时依然很久

              增量标记：将GC变成可中断

              中断后恢复的方法：

              三色标记法：

              ​ 白色：无活动；灰色：自身标记、成员未标记；黑色：已全标记

              ​ 每次中断恢复时就从灰色标记开始执行

              写屏障：

              ​ 白色对象被引用后还未标记，可能在下一轮被回收的问题

              ​ 强制让被黑色引用的白色变灰色

              懒清理

              如果内存足够执行代码，就延迟清理，也不需要全部清理，可以慢慢清

              增量标记和懒清理减少了STW的时间，但是没有减少主线程的总暂停时间

              并发回收

              用辅助线程GC，主线程不挂起

              需要有额外的读写锁机制控制引用关系的变化

              老生代主要使用并发标记、并发清理，同时用增量的方式分批执行

              9、内存泄漏

              无引用关系的对象没有被GC，则为内存泄漏

              常见的内存泄漏

              1、不正当闭包：有对其他作用域变量的引用未清除，造成内存泄漏

              2、隐式全局变量：没有声明，默认挂载全局；在函数中使用this产生隐式全局变量

              3、游离DOM引用：移除节点时释放缓存的引用。解决：将父节点和子节点的引用全都置空

              4、定时器clearInterval、clearTimeout、clearAnimationFrame

              5、事件监听器：解决：组件销毁时清除监听器

              6、监听者模式：组件销毁时清除处理函数

              export default {
  created() {
    eventBus.on("test", this.doSomething)
  },
  beforeDestroy(){
    eventBus.off("test", this.doSomething)
  },
  methods: {
    doSomething() {
      // do something
    }
  }
}
