docker

Docker Hub是官方提供的镜像仓库

常用命令

  • 启动docker：service docker start

  • 拉取mongo镜像：docker pull mongo

    • 默认端口27017

  • 启动镜像：docker run -d –name some-mongo -p 10050:27017 mongo

    • -d代表后台运行，跑在服务器的10050端口，映射到容器内的27017端口

  • 查看已启动的镜像docker ps

  • 查看本地所有镜像：docker images

  • 停止运行：docker stop

  • 删除镜像：docker rmi，-f表示强制删除，尽量先删除容器再删除镜像

  • 列举运行中的Docker容器：docker ps

    • -a列表所有的容器，-f过滤，-q只列表容器的id

  • 查看docker的版本信息：docker version，查看Docker的系统信息：docker info

  • 查看日志：docker logs -f xxx

  • 进入容器终端：docker exec -it redis-test /bin/bash

docker-compose

不需要手动运行多个服务
compose将容器分为三层，yml是一个工厂，工程下有多个服务，每个服务定义可以多个容器实例

  1. 配置docker-compose.yml文件
    build：Compose 会利用它自动构建镜像，该值可以是路径，也可以是对象，用于指定 Dockerfile 参数
    image：指定服务所使用的镜像
    ports：对外暴露的端口定义，和 expose 对应
  2. 构建、启动容器：docker-compose up -d
  • 常用命令
  • 列出所有运行容器：docker-compose ps
  • 查看服务日志输出：docker-compose logs
  • 启动指定服务已存在的容器：docker-compose start eureka

常见场景和设计方法
内嵌：数组内嵌，减少了关联查询，适合单类描述的属性和不经常变化的属性，如果是拓展属性或者需要单独访问的则不合适
父子引用：
父引用：存在一对多的情况，放在同一文档中，以数组形式存放
子引用：存在一对多的情况，反向引用，例如文章里的评论
数据内容非常多，或者需要单独访问，则不适合内嵌
反范式：
范式指特定的用法，反范式指不走寻常路
是否有提升性能的空间，数据提升是否庞大
先考虑读写比，再考虑反范式
双向关联的场景及设计原则
