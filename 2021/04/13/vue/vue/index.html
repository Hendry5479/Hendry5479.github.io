    vue实例

    let app = new Vue({
  el: '#app',
  data: {
    counter: 0
  },
  methods: {
    increment() {this.counter++}
  }
})

    • 细节：

      • el：string或者HTML元素
      • data：组件中的data必须是函数

    • Vue的MVVM

    • Vue生命周期

      • new Vue() -> create() -> Mount() -> Update() -> Destroy()

    基础语法

    Mustache

    • 双大括号

      v-once

      表示元素或组件只渲染一次，不希望界面随意改变

      v-html

      按照HTML格式进行解析，并且显示对应的内容

      v-text

      显示数据

      v-pre

      跳过该元素和子元素的编译，显示原本语法

      v-cloak

      遮罩，未编译的标签设置特定样式

      v-bind

    • 绑定属性，向组件传递props

    • 语法糖：简写形式 :

      v-bind绑定class

    • 对象语法

      • 与普通类同时存在，不冲突
      • 过于复杂的可以放在methods或者computed中
        :class="{'active': isActive}"
class="title" :class="{'active': isActive, 'line': isLine}"
:class="classes" （计算属性）

    • 数组语法

      :class="['active']">
:class=“[‘active’, 'line']"

      v-bind绑定style

      :style="{color: currentColor, fontSize: fontSize + 'px'}"
v-bind:style="[baseStyles, overridingStyles]"

    计算属性

    每个计算属性都包含一个getter和一个setter(不常用)
    methods和computed的区别：
    computed会缓存

    v-on

    • 缩写为@，预期是function 或者obj
    • 参数event
    • 如果方法不需要额外参数，()可以不添加
    • 如果需要同时传入某个参数，同时需要event时，可以通过$event传入事件
    • v-on修饰符
      • .stop：调用event.stopPropagation()，阻止行为
      • .prevent：调用preventDefault()
      • .{keyCode | keyAlias}：只有事件从特定键触发时才回调
      • .native：监听组件根元素的原生事件
      • .once：只触发一次回调

    v-if

    <div v-if="score >= 90">S</div>
<div v-else-if="score >= 80">A</div>
<div v-else-if="score >= 70">B</div>
<div v-else>C</div>

    Vue在进行DOM渲染时，出于性能考虑，会尽可能的复用已经存在的元素，而不是重新创建新的元素。
    如果不希望Vue出现类似重复利用的问题，可以给对应的input添加key，并且需要保证key的不同

    v-show

    决定一个元素是否渲染
    v-if：压根不会有对应的元素在DOM中
    v-show：display为none
    频繁切换的使用v-show

    v-for

    遍历数组

    • 如果在遍历的过程中不需要使用索引值：v-for=”movie in movies”
    • 如果需要索引值：v-for=(item, index) in items

      遍历对象

      <li v-for="(value, key, index) in info">
</li>
info: {...}

    key属性

    v-for时推荐添加:key属性，用于提高更新虚拟DOM的效率
    Vue中包含了一组观察数组编译的方法，使用它们改变数组也会触发视图的更新

    v-model

    实现表单控件和数据的双向绑定

    <input type="text" v-model="message">
<h2>{{message}}</h2>
    • v-model原理
      <input type="text" v-model="message">
等同于
<input type="text" v-bind:value="message" v-on:input="message = $event.target.value">
      radio, checkbox, select

    值绑定

    动态给value赋值，v-bind

    修饰符

    • lazy修饰符

      • 默认情况下，v-model在input事件中会同步输入框的数据
      • lazy修饰后只在失去焦点或者回车时更新
      • v-model.lazy

    • number修饰符

      • 默认下，字母或者数字都会当做字符串处理
      • 如果希望做当属字处理，则写成 v-model.number

    • trim修饰符

      • 如果输入的内容首尾空格居多，使用trim

    注册组件

    组件必须挂在在某个Vue实例中，否则不生效

    // 创建组件构造器，传入模板，
const cpn = Vue.extend({
  template: `xxx`
})
// 将组件构造器注册为组件，并命名
Vue.component('cpn', cpn)

    全局组件和局部组件

    Vue.component注册的是全局组件
    如果在某个Vue实例的components:{}中注册，则是局部组件
    父子组件之间，子组件只能在父组件中识别，否则会被浏览器忽略

    模板的分离写法

    使用script或者template标签分离html内容，指定id，然后挂在到组件上

    组件的数据存放

    组件对象也有一个data属性，但是只能是函数，函数返回一个对象，内部保存数据
    为什么必须是函数：
    Vue让每个组件对象返回一个新对象，因为如果是同一个对象，组件多次使用会相互影响

    父子间通信

    通过props向子组件传递数据 pass props
    通过事件向父组件发送消息 emit events

    props

    值类型：字符串数组或者对象
    需要数据验证时，就需要对象写法
    支持的类型：String、Number、Boolean、Array、Object、Date、Function
    另外还支持自定义类型验证

    子级向父级传递

    传递方式：自定义事件
    v-on不仅可以监听DOM事件，还可以用于自定义事件
    流程：
    子组件通过$emit()触发事件
    父组件通过v-on监听子组件事件

    父子组件访问方式：$children

    父访问子：$children或者$refs
    子访问父：$parent

    父子组件的访问方式：$refs

    通过$children访问子组件时，是数组类型，通过索引值访问子组件，难以确定子组件和索引的关系
    通过$refs能明确拿到某个特定组件
    $refs和ref一起使用，用ref给子组件绑定ID，通过this.$refs.ID可以访问到该组件

    父子组件的访问方式：$parent

    非父子组件通信的方式
    V1中通过$dispatch和$broadcast完成
    V2中通过中央事件总线完成
    后续通过Vuex完成状态管理

    el和template区别

    el用于指定Vue要管理的DOM，可以帮助解析其中的指令、事件监听等等。
    而如果Vue实例中同时指定了template，那么template模板的内容会替换掉挂载的对应el的模板
