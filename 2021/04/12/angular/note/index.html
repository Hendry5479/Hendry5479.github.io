Angular-Note

angular
front
Publish Date:   2021-04-12

ng常用命令

ng new project-name
ng serve –open 构建应用并监听文件
–open 标志会打开浏览器
–host 0.0.0.0 指定host地址
ng g component name

import { Component, OnInit } from '@angular/core';

@Component({
  selector: 'app-heroes',
  templateUrl: './heroes.component.html',
  styleUrls: ['./heroes.component.css']
})

导入 Component 符号，并为组件类加上 @Component 装饰器，用于为该组件指定 Angular 所需的元数据
里面是CLI 自动生成了三个元数据属性
selector— 组件的选择器（CSS 元素选择器）
templateUrl— 组件模板文件的位置。
styleUrls— 组件私有 CSS 样式表文件的位置。

UppercasePipe 格式化

<h2>{{hero.name | uppercase}} Details</h2>
uppercase 位于管道操作符（ | ）的右边，用来调用内置管道 UppercasePipe
管道 是格式化字符串、金额、日期和其它显示数据的好办法

双向绑定

<input [(ngModel)]="hero.name" placeholder="name"/>

[(ngModel)] 是 Angular 的双向数据绑定语法
这里把 hero.name 属性绑定到了 HTML 的 textbox 元素上

ngModel 在默认情况下是不可用，它属于一个可选模块 FormsModule，必须添加此模块才能使用该指令
在 AppModule 中，从 @angular/forms 库中导入 FormsModule， 添加到 @NgModule 元数据的 imports 数组

@NgModule

Angular 需要知道如何把应用程序的各部分组合到一起，这些信息被称为元数据（metadata）

有些元数据位于 @Component 装饰器中，你会把它加到组件类上。
另一些关键性的元数据位于 @NgModule 装饰器中
最重要的 @NgModule 装饰器位于顶层类 AppModule 上

声明 Component

Component 需要声明在AppModule的 @NgModule.declarations 数组中

*ngFor

<li *ngFor="let hero of heroes">

*ngFor 是一个复写器（repeater）指令，它会为列表中的每项数据复写它的宿主元素

  • 就是 *ngFor 的宿主元素，

    组件样式

    @Component 元数据中指定的样式和样式表都是局限于该组件的

    click 事件绑定

    <li *ngFor="let hero of heroes" (click)="onSelect(hero)">
    click 外面的圆括号会让 Angular 监听这个

  • 元素的 click 事件。 当用户点击
  • 时，Angular 就会执行表达式 onSelect(hero)。

    *ngIf

    <div *ngIf="selectedHero">
    selectedHero为undefined时不进行显示，避免报错

    CSS 类绑定机制

    [class.classname]="some-condition" 
eg: [class.selected]="hero === selectedHero"

    @Input()

    import { Component, OnInit, Input } from '@angular/core';
@Input() hero: Hero;

    从hero-com传入某个hero信息到hero-detail中
    hero 必须是带有 @Input() 装饰器的输入属性，因为外部的 HeroesComponent 将会绑定到它
    <app-hero-detail [hero]=”selectedHero”>
    [hero]=”selectedHero” 是属性绑定语法，单向数据绑定，selectedHero 属性绑定到目标元素的 hero 属性

    创建服务

    用Angular 的依赖注入机制将service注入组件构造函数中
    创建 HeroService：ng generate service hero，生成如下：

    import { Injectable } from '@angular/core';
@Injectable({
  providedIn: 'root',
})
export class HeroService {
  constructor() { }
}

    @Injectable() 装饰器会接受该服务的元数据对象，就像 @Component() 对组件类的作用一样。
    注入器是一个对象，负责当应用要求获取它的实例时选择和注入该提供者。

    注入 HeroService

    在component中：
    constructor(private heroService: HeroService) { }
    在创建组件时，依赖注入系统把service设置为组件的单例对象

    可观察（Observable）数据

    HeroService.getHeroes() 是异步函数，需要使用HttpClient.get获取数据，并返回Observable对象
    Observable 是 RxJS 库中的一个关键类

    使用RxJS的of( )模拟服务器返回数据，hero.service中：

    import { Observable, of } from 'rxjs';
getHeroes(): Observable<Hero[]> {
  return of(HEROES);
}

    of(HEROES) 会返回一个 Observable<Hero[]>，它会发出单个值，这里是一个英雄数组

    在 Component 中订阅Service

    getHeroes(): void {
  this.heroService.getHeroes()
      .subscribe(heroes => this.heroes = heroes);
}

    在模板中绑定的组件属性必须是公共属性

    添加AppRoutingModule

    ng generate module app-routing –flat –module=app
    –flat 把文件放进src/app 中，而不是单独的目录中。
    –module=app 告诉 CLI 把它注册到 AppModule 的 imports 数组中

    Module代码：

    import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { HeroesComponent } from './heroes/heroes.component';

const routes: Routes = [
  { path: 'heroes', component: HeroesComponent }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }

    导入 RouterModule 和 Routes，以便该应用具有路由功能
    Routes属性有两个参数，path和component

    RouterModule.forRoot()
    @NgModule 元数据会初始化路由器，并监听地址变化
    forRoot() 会提供路由所需的服务提供者和指令，还会基于浏览器的当前 URL 执行首次导航

    paramMap 是从 URL 提取的路由参数值的字典

    启用 HTTP 服务

    HttpClient 是 Angular 通过 HTTP 与远程服务器通讯的机制
    把HttpClient添加到根模块 AppModule 中

    import { HttpClientModule } from '@angular/common/http';

@NgModule({
  imports: [
    HttpClientModule,
  ],
})

    模拟数据服务器 In-memory Web API

    npm install angular-in-memory-web-api –save

    在 AppModule 中导入 HttpClientInMemoryWebApiModule 和 InMemoryDataService 类

    将 WebApiModule 添加到 AppModule 的 imports 数组中，并以 InMemoryDataService 为参数进行配置

    HttpClientInMemoryWebApiModule.forRoot(
  InMemoryDataService, { dataEncapsulation: false }
)

    forRoot() 配置方法接收一个 InMemoryDataService 类来初始化内存数据库

    模拟Http请求

    hero.service.ts代码如下：

    import { HttpClient, HttpHeaders } from '@angular/common/http';
constructor(private http: HttpClient,) { }

// 将 RxJS 的 of( ) 模拟转换成使用 HttpClient ，两者都返回Observable<Hero[]>
getHeroes(): Observable<Hero[]> {
  return of(HEROES);
}

getHeroes(): Observable<Hero[]> {
  return this.http.get<Hero[]>(this.heroesUrl)
}

    所有的 HttpClient 方法都会返回某个值的 RxJS Observable

    HttpClient.get() 返回响应数据
    get() 默认情况把响应体当做无类型的 JSON 对象进行返回。 如果指定了可选的模板类型 <Hero[]>，就会给返回你一个类型化的对象。

    API 可能在返回对象中深埋着你想要的数据，要借助 RxJS 的 map() 对结果处理

    错误处理

    捕获错误使用 RxJS 的 catchError() 建立对 Observable 结果的处理管道（pipe）
    /hero.service.ts代码如下：

    import { catchError, map, tap } from 'rxjs/operators';

getHeroes( ): Observable<Hero[]> {
  return this.http.get<Hero[]>(this.heroesUrl)
    .pipe(
      catchError(this.handleError<Hero[]>('getHeroes', [ ]))
    );
}

    catchError( ) 拦截失败的 Observable，把错误对象传给错误处理器

    handleError通用化

    不直接处理错误，而是返回给 catchError 一个错误处理函数

    private handleError<T>(operation = 'operation', result?: T) {
  // 匿名函数
  return (error: any): Observable<T> => {
    console.error(error); // log to console instead
    this.log(`${operation} failed: ${error.message}`);
    return of(result as T);
  };
}

    窥探 Observable

    使用 RxJS 的 tap( ) 查看 使用 传出Observable 的值，但不会改变值本身
    hero.service.ts代码如下：

    getHeroes( ): Observable<Hero[ ]> {
  return this.http.get<Hero[]>(this.heroesUrl)
    .pipe(
      tap(_ => this.log('fetched heroes')),
      catchError(this.handleError<Hero[]>('getHeroes', []))
    );
}

    通过 id 获取英雄

    hero.service.ts代码如下：
getHero(id: number): Observable<Hero> {
  const url = `${this.heroesUrl}/${id}`;  // 构建url

  return this.http.get<Hero>(url).pipe(
    tap(_ => this.log(`fetched hero id=${id}`)),
    catchError(this.handleError<Hero>(`getHero id=${id}`))

}

    删除某个英雄

    heroes.component.ts代码如下：

    delete(hero: Hero): void {
  // 去掉被删除数据后的列表
  this.heroes = this.heroes.filter(h => h !== hero);
  this.heroService.deleteHero(hero).subscribe();
}

hero.service.ts
deleteHero(hero: Hero | number): Observable<Hero> {
  const id = typeof hero === 'number' ? hero : hero.id;
  const url = `${this.heroesUrl}/${id}`;

  return this.http.delete<Hero>(url, this.httpOptions).pipe(
    tap(_ => this.log(`deleted hero id=${id}`)),
    catchError(this.handleError<Hero>('deleteHero'))
  );
}

    根据名字搜索

    当用户在搜索框中输入名字时，会不断发送HTTP 请求，目标是发出尽可能少的必要请求
    hero.service.ts

    searchHeroes(term: string): Observable<Hero[]> {
  if (!term.trim()) {
    // if not search term, return empty hero array.
    return of([]);
  }

  return this.http.get<Hero[]>(`${this.heroesUrl}/?name=${term}`).pipe(
    tap(x => x.length ?
       this.log(`found heroes matching "${term}"`) :
       this.log(`no heroes matching "${term}"`)),
    catchError(this.handleError<Hero[]>('searchHeroes', []))
  );
}

    展示搜索结果的列表
    hero-search.component.html

    <div id="search-component">
  <h4><label for="search-box">Hero Search</label></h4>

  <input #searchBox id="search-box" (input)="search(searchBox.value)" />

  <ul class="search-result">
    <li *ngFor="let hero of heroes$ | async" >
      <a routerLink="/detail/{{hero.id}}">
        {{hero.name}}
      </a>
    </li>
  </ul>
</div>

    $ 是一个约定，表示 heroes$ 是一个 Observable 而不是数组
    由于 *ngFor 不能直接使用 Observable，要使用管道字符（|），后面跟着 async表示 AsyncPipe 管道，它会自动订阅 Observable，组件就不再需要订阅heroes$

    RxJS Subject 类型的 searchTerms

    private searchTerms = new Subject<string>( );
search(term: string): void {
  this.searchTerms.next(term);
}

    Subject 既是Observable对象的数据源，本身也是 Observable
    通过调用next(value) 往 Observable 中推送一些值

    串联 RxJS 操作符

    ngOnInit( ) 往 searchTerms 的处理管道中加入 RxJS 操作符，缩减对 searchHeroes( ) 的调用次数，并返回一个搜索结果的可观察对象（Hero[ ] ）

    this.heroes$ = this.searchTerms.pipe(
  // 传出最终字符串之前，debounceTime(300) 将会等待直至300内无新事件
  debounceTime(300),
  // 只在过滤条件变化时发送请求
  distinctUntilChanged(),
  // 取消丢弃旧的数据，保留最新的
  switchMap((term: string) => this.heroService.searchHeroes(term)),
);
