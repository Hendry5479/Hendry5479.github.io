7.清除浮动及优点

父元素高度塌陷，因为子浮动而且父不设高度
深入剖析原因

清除浮动
1.父设置高度，不能响应式
2.父overflow: hidden; zoom:1; 超出部分被隐藏，需要注意
3.浮动元素后面加空标签，clear: both; height: 0; overflow: hidden; 增加空标签
4.父overflow: auto; 内部宽高超过父div时，出现滚动条
5.万能清除法

.father:after {
  content: '';
  clear: both;
  display: block;
  height: 0;
  overflow: hidden;
  visibility: hidden;
}

8.定位position

relative：不脱离，相对自己
absolute：脱离，相对父级
fixed：脱离，相对窗口
static：默认

9.子元素在父元素居中

水平居中
1.子父宽度固定，子设置margin:auto，并且不能设置浮动
2.子父宽度固定，父设置text-align: center，子设置displat:inline-block;并不设置浮动

水平垂直居中
子绝父相，子top，left50%，margin-top, left减去子宽高的一半
子绝父相，子top，left50%，transform: translate(-50%, -50%)
子绝父相，子上下左右全0，margin：auto
父display:table-cell; vertical-align: middle，子margin: auto;
子相，top, left50%, transform: translate(-50%, -50%)
父弹性盒子，display: flex; justify-content: center; align-item: center;

11.元素垂直居中

1.设置子元素和父元素行高一致
2.子元素设为inline-block，加上vertical-align: middle;
3.已知父高度，子相，子top50%，transform： translateY(-50%)
4.不知父高度，子绝父相，子top50%，transform： translateY(-50%)
5.创建隐藏节点，设置height
6.父display:table, 子display:table-cell，vertical-align: middle
7.父添加伪元素
8.父弹性盒子，子align-self:center;

14.图片加载优化

1.懒加载，不可视区域添加滚动事件，判断图片位置和浏览器顶端和页面的距离，如果前者小于后者，优先加载
2.预加载，将当前展示的前后两张优先下载
3.雪碧图csssprite或者svgsprite

15.行内元素/块级元素

行内：相邻的行内元素会排列同行，宽高无效 span
块级：独占一行，宽高有效，div
可变：根据上下文决定块级还是内联

块级：div,h,p,ul,ol,table,from, menu
行内元素：a,br,i,imginput,select,span,sub,sup,textarea
可变：button,del,iframe,ins

16.浏览器的标准模式和怪异模式

怪异：浏览器根据自己的方式解析
